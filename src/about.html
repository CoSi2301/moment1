Uppgiftsbeskrivning

Här ska du skapa en grundstruktur för webbutveckling kompletterat med automatisering av uppgifter. Det ska vara för en webbplats som innehåller HTML, CSS, JavaScript och bilder. Du skall använda NodeJs för att automatisera uppgifterna. Du kan använda exempelvis Parcel, Gulp eller annat system för detta. Installerade paket ska finnas med i en package.json-fil. Det ska gå att återställa din applikation med npm install. De uppgifter som skall automatiskeras är:

  • Minifiering av CSS. (Vi kommer även använda transpilering av SCSS/SASS till CSS i senare moment i kursen)
    • Parcel minifierar CSS automatiskt när man kör build. Har för närvarande sex .css-filer som blir minifierade och sammanslagna till en.
  • Transpilering och minifiering av JavaScript.
    • Parcel minifierar Javascript automatiskt när man kör build. Har för närvarande en .js-fil som blir minifierad och transpilerad (Se nedan).
  • Minifiering av HTML.
    • Parcel minifierar HTML automatiskt när man kör build. Har för närvarande tre .html-filer som blir minifierade.
  • Optimering av bilder - skall ske automatiskt vid uppdatering av bilder. Använd moderna bildformat (exempelvis WebP eller AVIF) och passande inställningar för dessa.
    • Optimerar bilder med sharp. När jag ändrade grundinställningen för kvaliteten på bilderna (80 default) till 70 så sparar den nya bilder som jpeg. Mina bilder hade filändelsen jpg och jag fattar verkligen inte hur jag kan styra från jpeg till jpg.
      Har "löst" det genom att döpa om originalen till jpeg. Utöver att anpassas repsonsivt utifrån skärmstorlek så skapas optimerade webp-versioner i picture-taggen.
  • Bakåtkompabilitet i genererade filer, så att exempelvis JavaScript transpileras så att dessa fungerar i så många webbläsare som möjligt.
    • Använder Browserlists förinställning: > 0.5%, last 2 versions, Firefox ESR, not dead.
      Om jag förstått rätt så träffar vi webbläsare över 0.5% av marknaden , dessa webbläsares 2 senaste versioner , Firefox ESR (Firefox långtidssupport [kort och koncist en version som uppdateras långsammare]) och utesluter "döda" webbläsare.
  • Kopiering av arbetsfiler till mapp för publicering med optimerade filer.

Skapa en demowebbplats som innehåller HTML, CSS och JavaScript. Sidan skall innehålla en huvudmeny med stöd för ett godtyckligt antal menyalternativ. Sidan ska naturligtvis var fullt responsiv. Webbplatsen skall bestå av minst tre undersidor:

  • Startsida med info om uppgiften
  • Undersida för bilder med minst fem stycken fotografier
  • En undersida för publicering där du beskriver din skapade arbetsprocess (se nedan).

Filerna skall vara väl strukturerade, med underkataloger för CSS, JavaScript och bilder. Det ska finnas skiljd filstruktur mellan arbetsfiler och publiceringsfiler.

Börjat med att ange lite grundläggande saker för automatiseringen såsom Parcel och skapa en repo på Github. Denna repo har både en Main branch och en branch kallad develop som används. Sedan hämtar netlify informationen från Github och uppdaterar när main branch blir uppdaterad.

Sedan skapades de grundläggande filerna som krävs för uppgiften.Ett väldigt grundläggande utseende har sedan tillämpats. Därefter har bilder fotats och lagts till. Med hjälp av sharp så skapas automatiskt nu .webp-versioner av alla bilderna i respektive storlek för satta media queries.
